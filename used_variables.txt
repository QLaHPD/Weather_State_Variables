FuXi Input Variables
====================

This file lists the input variables needed by the current FuXi-style models in this
repo.

The dynamic weather input uses 2 time steps:
- x: [B, 2, 70, H, W]

The 70 dynamic channels are the official FuXi variables, in the exact order documented
by the original repository.

2. 2D Surface Variables
-----------------------
These are the last 5 channels of the 70-channel dynamic input.

Exact channel order:
- T2M: 2-meter temperature
- U10: 10-meter u wind
- V10: 10-meter v wind
- MSL: mean sea-level pressure
- TP: total precipitation


3. Static Auxiliary Channels
----------------------------
The checkpoint also uses 5 static channels:
- static_features: [B, 5, H, W]

Important note:
- Public FuXi/FuXi-ENS descriptions usually name 4 geographical inputs:
  - orography
  - land-sea mask
  - latitude
  - longitude
- But the actual short.onnx checkpoint stores 5 static tensor channels.
- From direct inspection of the ONNX weights, the 5 channels used by this checkpoint are:

1. Land-sea mask or land fraction map
- Inferred from the ONNX tensor statistics.
- Values are in [0, 1].

2. Orography / terrain height map
- Inferred from the ONNX tensor statistics.
- Stored as a normalized continuous field.

3. cos(latitude)
- Verified directly from the ONNX tensor values.

4. cos(longitude)
- Verified directly from the ONNX tensor values.

5. sin(longitude)
- Verified directly from the ONNX tensor values.

So the model effectively uses:
- 65 upper-air channels
- 5 surface channels
- 5 static channels


4. Time Embedding Input
-----------------------
The model also requires a separate temporal embedding input:
- temb: [B, 12]

This is not part of the 70 weather channels or the 5 static channels.


5. Summary
----------
To run the model, you need:
- x with shape [B, 2, 70, H, W]
- the 70 channels in the exact order listed above
- static_features with shape [B, 5, H, W] or a broadcast-compatible equivalent
- temb with shape [B, 12]
